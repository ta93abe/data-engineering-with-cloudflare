## 1. コンピューティング

### 1.1 Cloudflare Workers

**ステータス**: ✅ GA（一般公開）

**概要**:
JavaScriptランタイムでのサーバーレスコンピューティング。V8エンジンベースで世界中のデータセンターで実行。

**データ基盤での活用方法**:
- データ取り込みAPIの実装
- ETLパイプラインの実装
- データ変換・クレンジング処理
- リアルタイムデータ処理
- APIゲートウェイとしての利用

**2025年の新機能**:
- **フルスタック対応**: 静的アセットのサポートにより、単一Workerでフルスタックアプリケーションをデプロイ可能
- **フレームワーク拡充**: React Router v7、Astro、Hono、Vue.js、Nuxt、SvelteKit対応。Next.js、Angular、SolidJSは2025年Q2対応予定
- **Python Workers**: コールドスタートパフォーマンスが向上し、初期化時間が短縮
- **ローカル開発改善**: `wrangler dev`でTLS必須のリモートデータベースに直接接続可能

**制限値**:
- CPU時間: 50ms（Free）/ 30秒（Paid）
- メモリ: 128MB
- リクエストサイズ: 100MB

**価格**:
- Free: 100,000リクエスト/日
- Paid: $5/月 + $0.30/百万リクエスト

**データ基盤での推奨度**: ⭐⭐⭐⭐⭐（必須）

**ステータス**: 🚧 近日公開（2025年6月予定）

**概要**:
フルLinuxサポート、マルチコア、ギガバイト単位のメモリを持つコンテナをWorkersインターフェースでデプロイ可能。

**データ基盤での活用方法**:
- 重量級のデータ処理ワークロード
- 既存のDockerコンテナをエッジで実行
- カスタムバイナリやツールチェーンの実行
- 長時間実行バッチ処理
- GPU不要の機械学習推論

**期待される特徴**:
- マルチコアCPU対応
- GB単位のメモリ
- フルLinux環境
- Workersと同じグローバルネットワーク

**データ基盤での推奨度**: ⭐⭐⭐⭐（公開後に評価）

### 1.4 Workers for Platforms

**ステータス**: ✅ GA（一般公開）

**概要**:
マルチテナントSaaSアプリケーション構築のためのプラットフォーム。顧客ごとに分離されたWorkerを実行。

**データ基盤での活用方法**:
- マルチテナントデータ処理プラットフォーム
- 顧客ごとのカスタムETLロジック実行
- テナント分離されたデータパイプライン
- エンドユーザー向けスクリプト実行環境
- データ変換ルールのカスタマイズ

**主な機能**:
- **分離とセキュリティ**: 各顧客のコードは独立したWorkerサンドボックスで実行
- **大規模スケール**: 標準Workersの500スクリプト制限を超え、数百万のユーザースクリプトを管理可能
- **トラフィック管理**: テナントごとの制限設定、リクエストごとの課金計測
- **カスタムドメイン**: 顧客ブランドのドメイン自動対応
- **ストレージ公開**: D1、KV、R2、VectorizeをAPIキー漏洩なく拡張機能に公開

**コスト効率**:
- アイドル時 $0
- 実行時のみ課金（従来の時間ベースモデルより約70%削減）

**データ基盤での推奨度**: ⭐⭐⭐⭐（マルチテナント用途）

### 1.5 Workflows

**ステータス**: ✅ GA（一般公開、2025年）

**概要**:
耐久性のある実行エンジン。複数ステップのアプリケーションを自動リトライ、状態永続化、長時間実行（分、時間、日、週単位）で構築可能。

**データ基盤での活用方法**:
- 複雑なETLパイプラインのオーケストレーション
- 長時間実行のデータ処理ワークフロー
- マルチステップデータ変換
- イベント駆動型データパイプライン
- AIエージェントワークフロー
- 外部システムとの連携フロー

**2025年の新機能**:
- **Python対応**: TypeScriptに加えてPythonをサポート（データパイプライン、AI/ML用途に最適）
- **テスト改善**: より良いテスト体験を提供
- **ストレージ課金**: 状態保存のストレージ課金は2025年9月15日まで無料

**アーキテクチャ**:
- Durable ObjectsとSQLiteストレージを基盤
- 各ステップの状態を自動永続化
- 障害時も正確に中断箇所から再開

**ユースケース**:
- Eコマース注文処理
- AIエージェントワークフロー
- データパイプライン
- イベント駆動マルチステップ処理

**データ基盤での推奨度**: ⭐⭐⭐⭐⭐（オーケストレーション必須）

## 2. ストレージ・データベース

### 2.1 D1 Database

**ステータス**: ✅ GA（一般公開）

**概要**:
SQLiteベースのサーバーレスSQLデータベース。グローバルクエリを高速実行。

**データ基盤での活用方法**:
- トランザクションデータの保存
- リレーショナルデータ管理
- メタデータストア
- 集計データマート
- ユーザープロファイル管理

**2025年の新機能**:
- **パフォーマンス向上**: エンドツーエンドのWorker APIリクエストレイテンシが40-60%削減（冗長なネットワークラウンドトリップを排除）
- **リードレプリカ（ベータ）**: スケーラビリティ向上のためのリードレプリカ対応
- **Free Tier制限適用**: 2025年2月10日からWorkers Freeプランのアカウントに対してFree Tier制限を適用開始

**制限値**:
- データベースサイズ: 10GB（有料プラン）
- SQLiteの機能セットに準拠

**データ基盤での推奨度**: ⭐⭐⭐⭐⭐

---

### 2.2 Workers KV

**ステータス**: ✅ GA（一般公開）

**概要**:
低レイテンシのキー・バリューストレージ。最終的整合性モデル。

**データ基盤での活用方法**:
- データキャッシング層
- セッション管理
- 設定データの配信
- APIキー管理
- 頻繁に参照されるメタデータ

**2025年の新機能**:
- **自動プロビジョニング**: Wranglerがデプロイ時にKVリソースを自動作成

**特徴**:
- エッジでキャッシュされる超高速読み取り
- 数千RPS以上の読み取りに最適
- グローバル分散

**制限値**:
- キー: 最大512バイト
- バリュー: 最大25MB
- 書き込みは最終的整合性

**価格**:
- ストレージ: $0.50/GB/月
- 読み取り: 最初の1,000万リクエスト無料、以降$0.50/1,000万
- 書き込み: $5/1,000万

**データ基盤での推奨度**: ⭐⭐⭐⭐⭐

---

### 2.3 R2 Object Storage

**ステータス**: ✅ GA（一般公開）

**概要**:
S3互換のオブジェクトストレージ。エグレス料金無料。

**データ基盤での活用方法**:
- データレイクの構築
- 大容量ファイルストレージ
- ログアーカイブ
- バックアップストレージ
- メディアアセット保存
- ETL中間データ保存

**2025年の新機能**:
- **R2 SQL（オープンベータ、2025年9月）**: Apache Icebergテーブルに対してSQLクエリを実行可能
- **集計関数対応**: COUNT(*)、SUM()、AVG()、MIN()、MAX()などの標準SQL関数
- **GROUP BY、HAVING対応**: 集計クエリの高度化
- **スキーマ検出コマンド**: Apache Icebergテーブルのスキーマ自動検出
- **イベント通知（オープンベータ）**: R2バケット内のデータ変更時にQueueへメッセージ送信
- **R2 Data Catalog（ベータ、2025年4月）**: Apache Icebergデータカタログ機能をR2バケットに直接統合

**特徴**:
- S3 API互換
- エグレス帯域幅が無料（大幅なコスト削減）
- GCS（Google Cloud Storage）からの移行サポート

**制限値**:
- オブジェクトサイズ: 最大5TB（マルチパートアップロード）
- バケット名はグローバルユニーク

**価格**:
- ストレージ: $0.015/GB/月
- Class A操作（書き込み）: $4.50/100万リクエスト
- Class B操作（読み取り）: $0.36/100万リクエスト
- エグレス: **無料**

**データ基盤での推奨度**: ⭐⭐⭐⭐⭐（コスト効率最高）

### 3.2 AI Gateway

**ステータス**: ✅ GA（一般公開）

**概要**:
AIアプリケーションのための統合制御プレーン。20以上のAIプロバイダーをサポート。

**データ基盤での活用方法**:
- AIリクエストのキャッシング（コスト削減）
- AI利用状況の監視・分析
- マルチプロバイダー管理
- コスト追跡と最適化
- AIデータパイプラインの可観測性
- レート制限・DLP（Data Loss Prevention）

**2025年の新機能**:
- **統一請求**: 複数プロバイダーの請求を一元化
- **セキュアキーストレージ**: APIキーの安全な保管
- **動的ルーティング**: 最適なAIモデルへのルーティング
- **DLPセキュリティ制御**: データ損失防止機能

**主な機能**:

#### キャッシング
- 同一リクエストをCloudflareのグローバルキャッシュから提供
- レイテンシ最大90%削減
- コスト大幅削減（重複API呼び出しを回避）
- テキスト・画像レスポンス対応
- リクエストごとのキャッシュ設定上書き可能
- `cf-aig-cache-status`ヘッダーでHIT/MISS判定

#### 可観測性
- 詳細なアナリティクス: リクエスト数、トークン数、コスト、エラー、パフォーマンス
- 全プロバイダー横断で追跡
- リクエスト・レスポンスの詳細ログ取得（デバッグ、コンプライアンス、分析）
- カスタムメタデータでログ・アナリティクスを強化（ユーザーID、チーム情報など）
- キャッシュ率をダッシュボードで確認

**データ基盤での推奨度**: ⭐⭐⭐⭐⭐（AI活用時必須）

## 4. データ処理・ETL

### 4.1 Pipelines

**ステータス**: 🔵 オープンベータ

**概要**:
ストリーミングデータプラットフォーム。イベント取り込み、SQL変換、R2へのApache IcebergテーブルまたはParquetファイル書き込み。

**データ基盤での活用方法**:
- リアルタイムデータ取り込み
- ストリーミングETLパイプライン
- データクレンジング・変換
- データウェアハウスへの投入
- イベントデータのアーカイブ
- ログ収集・処理

**2025年の新機能**:
- **SQL変換対応（2025年9月）**: ストリームをSQLで検証、フィルタ、変換、エンリッチ
- **Apache Iceberg対応**: R2 Data CatalogへApache Icebergテーブルとして書き込み
- **Parquet/JSON出力**: ファイル形式での出力もサポート
- **Arroyo統合**: CloudflareがArroyoを買収し、ストリーミング取り込み機能を強化

**主な機能**:

#### データ取り込み
- HTTPまたはWorkerバインディング経由で取り込み
- 単一パイプラインで最大100MB/秒

#### SQL変換
- ストリームとシンクをSQL変換で接続
- データの検証、フィルタ、変換、エンリッチを取り込み時に実行
- 現在はステートレス変換のみ対応
- **将来計画**: Arroyoのステートフル処理機能（集計、インクリメンタル更新マテリアライズドビュー、結合）

#### 出力先
- R2 Data CatalogへApache Icebergテーブル書き込み
- ParquetおよびJSONファイルとしてエクスポート

#### 信頼性
- Exactly-once保証でR2へ配信

**価格**:
- オープンベータ中は無料（R2ストレージ・操作は標準料金で課金）

**データ基盤での推奨度**: ⭐⭐⭐⭐⭐（ストリーミングETL必須）

- 静止画フレーム抽出
- 音声抽出
- ショート動画の加工

**2025年の新機能**:
- **制限拡大（2025年6月）**: 入力ファイルサイズ100MB（従来40MB）、出力動画時間1分（従来30秒）

- **音声抽出**: 動画から音声のみを抽出

**特徴**:
- Cloudflare Stream外の動画にも対応
- シンプルなURL形式でリクエスト
- HTML video要素で直接使用可能（プレイヤーライブラリ不要）

**価格（2025年11月1日から）**:
- Image Transformationsと同じ課金体系
- 静止画フレーム生成: 1変換としてカウント
- 動画最適化・音声抽出: 出力1秒あたり1変換
- ユニークな変換は月1回のみ課金

**ベータ期間**: 2025年Q3まで無料予定

**データ基盤での推奨度**: ⭐⭐⭐（動画処理時）

## 7. リアルタイム通信

- SFU（Selective Forwarding Unit）機能
- TURNリレー（NAT/ファイアウォール越えの通信）

**価格**:

### 8.2 Browser Rendering

**ステータス**: 🔵 オープンベータ

**概要**:

**データ基盤での活用方法**:

**特徴**:

#### 統合方法
1. **REST API**: 簡単なステートレスタスク（スクリーンショット、PDF、HTML抽出）

#### 対応ツール

- Playwright
- Playwright MCP
- Stagehand（AIエージェント）

#### Durable Objects連携
- ブラウザセッション永続化でパフォーマンス向上
- セッション再利用で起動時間削減
- 同時セッション数削減

**データ基盤での推奨度**: ⭐⭐⭐⭐（スクレイピング用途）
